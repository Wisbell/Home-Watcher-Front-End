About Home Watcher

The idea for Home Watcher came about when I finished researching the capabilities of my Raspberry Pi after using it on my front-end capstone at NSS. After deciding I could complete a minimum viable project in less than 2 weeks, I decided to start building. Below you can get a run down on how the app works, a list of github repos where each part of the project is located, a list of technologies used in the creation of the app, and my website where you can contact me (Westley Isbell)

How it Works
First: The motion sensor is calibrated and then triggered by movement
Second: A picture is taken with the camera module and saved to the Raspberry Pi file system
Third: The created picture file is then sent to AWS S3 for storage
Fourth: A collection is created on a Mongo database with the url from AWS and the date/time it was created
Fifth: The picture file is deleted from the Raspberry Pi file system
Sixth: A flag variable is reset to allow another picture to be taken if motion is sensed
Client-Side: After logging in a user can visit the recent route and see the most recent pictures generated by the Raspberry Pi system
